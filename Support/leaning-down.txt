Lib UI Lean-Down Plan
Principles
The data side (store, storage, mutations, coordinator, accessors, logging) is untouched.
Widget draw logic survives but is recast as plain functions, not registry entries.
Internal draw helpers (DrawStructuredAt, DrawOrderedEntries, NormalizeInteger, etc.) survive as private implementation detail.
The layout system, node tree, prepare/validate pipeline, and all caching built to serve them are removed entirely.
special.standaloneUI survives, simplified. runPass is inlined and removed as a public function.
Two structural helpers replace the layout system: lib.ui.verticalTabs and lib.ui.isVisible.

Phase 3 — Rewrite widgets as plain functions
Each widget becomes a plain function under lib.widgets. The WidgetTypes registry table goes away entirely. The widget files stop writing to WidgetTypes and instead write to public.widgets.

New function signature pattern:

function lib.widgets.NAME(imgui, uiState, alias, opts)
    -- read from uiState.view[alias]
    -- draw with imgui
    -- write with uiState.set(alias, value) on change
    -- manage cursor if needed
    -- return changed (bool)
end
opts is a plain table of optional parameters. No validation at call time — the caller is responsible. No node mutation, no caching, no prepare step.

lib.widgets.checkbox(imgui, uiState, alias, opts)

opts: label, tooltip

local label = (opts and opts.label) or alias
local current = uiState.view[alias] == true
local next, changed = imgui.Checkbox(label .. "##" .. alias, current)
if changed then uiState.set(alias, next) end
if opts and opts.tooltip and imgui.IsItemHovered() then
    imgui.SetTooltip(opts.tooltip)
end
return changed
Simple. No DrawStructuredAt needed. ImGui advances cursor naturally after Checkbox.

lib.widgets.dropdown(imgui, uiState, alias, opts)

opts: label, tooltip, values, displayValues, valueColors, controlWidth, controlGap

Internally uses NormalizeChoiceValue, ChoiceDisplay, DrawLabeledDropdownControl (internal helper surviving from dropdowns.lua), DrawWithValueColor, BeginCombo/EndCombo. The prepared _options list is built inline from opts.values at call time — no prepare step, no cache. For static value lists this is negligible cost. The list is typically 3–10 entries.

Returns changed.

lib.widgets.mappedDropdown(imgui, uiState, alias, opts)

opts: label, tooltip, getPreview, getOptions, getPreviewColor, controlWidth, controlGap

getPreview(view) and getOptions(view) receive uiState.view and return the preview string and options list. Options can be { value, label, color, onSelect } tables or plain values.

Same internal DrawLabeledDropdownControl helper. Returns changed.

lib.widgets.radio(imgui, uiState, alias, opts)

opts: label, tooltip, values, displayValues, valueColors, optionGap

Internally uses NormalizeChoiceValue, BuildOrderedChoiceEntries (internal helper surviving from radios.lua), DrawOrderedEntries. After drawing, sets cursor to (startX, startY + consumedHeight).

Returns changed.

lib.widgets.mappedRadio(imgui, uiState, alias, opts)

opts: label, tooltip, getOptions, optionGap

getOptions(view) returns option list. Otherwise same as radio. Returns changed.

lib.widgets.stepper(imgui, uiState, alias, opts)

opts: label, tooltip, min, max, step, fastStep, displayValues, valueColors, valueWidth, valueAlign

Internally uses BuildOrderedStepperEntries and DrawOrderedEntries — these survive as private functions within steppers.lua. Reads uiState.view[alias], normalizes, writes on change. Sets cursor after drawing. Returns changed.

lib.widgets.steppedRange(imgui, uiState, minAlias, maxAlias, opts)

opts: label, tooltip, min, max, step, fastStep, valueWidth, valueAlign

Two bind aliases. Internally builds combined entry list through existing BuildOrderedStepperEntries with geometryOwner = opts. Reads uiState.view[minAlias] and uiState.view[maxAlias]. Sets cursor. Returns changed.

lib.widgets.inputText(imgui, uiState, alias, opts)

opts: label, tooltip, maxLen, controlWidth, controlGap

Uses DrawLabeledInputControl (internal, simplified from current inputs.lua). Returns changed.

lib.widgets.packedDropdown(imgui, uiState, alias, store, opts)

opts: label, tooltip, noneLabel, multipleLabel, displayValues, valueColors, controlWidth, controlGap

store is passed explicitly so the function can resolve packed children via store.getPackedAliases(alias). This returns { { alias, label } }. The function reads individual bit aliases from uiState.view and writes them on change. Uses ClassifyPackedChoice, ApplyPackedChoiceSelection, ClearPackedChoiceSelection internally.

store needs a new method: store.getPackedAliases(alias) → list of { alias, label }. This is a thin wrapper over the existing _aliasNodes / _bitAliases machinery already in the store.

lib.widgets.packedRadio(imgui, uiState, alias, store, opts)

Same store dependency as packedDropdown. opts: label, tooltip, noneLabel, displayValues, valueColors, optionGap. Returns changed.

lib.widgets.packedCheckboxList(imgui, uiState, alias, store, opts)

opts: tooltip, valueColors, filterText, filterMode, slotCount

filterText and filterMode are plain strings passed directly rather than bound aliases. The filter is typically driven by module-local state, not persistent storage. Uses store.getPackedAliases(alias) for the item list, reads each item's value from uiState.view, writes on toggle. Returns changed.

lib.widgets.button(imgui, label, opts)

opts: tooltip, onClick, id

No storage alias. onClick(imgui) is called on press. id for ##id disambiguation. Returns changed.

lib.widgets.confirmButton(imgui, id, label, opts)

opts: tooltip, onConfirm, confirmLabel, cancelLabel

id is required and explicit — no more _imguiId from node prepare. Returns changed.

lib.widgets.separator(imgui)

No opts. Just imgui.Separator(). Returns nothing.

lib.widgets.text(imgui, text, opts)

opts: color, tooltip, alignToFramePadding

Direct imgui.Text or imgui.TextColored. No storage alias. Returns nothing.

Phase 4 — New lib.ui with structural helpers
Create a new field_registry/ui.lua (replacing the deleted one) that exposes only:

lib.ui.verticalTabs(imgui, opts)

opts: id, navWidth, height, tabs, activeKey

tabs is { { key, label, color } }. activeKey is the current active tab key passed in by the caller. Returns the new activeKey after rendering the sidebar selectables.

function lib.ui.verticalTabs(imgui, opts)
    local id = opts.id
    local navWidth = opts.navWidth or 180
    local height = opts.height or 0
    local tabs = opts.tabs or {}
    local activeKey = opts.activeKey

    imgui.BeginChild(id .. "##nav", navWidth, height, true)
    for _, tab in ipairs(tabs) do
        local selected = tab.key == activeKey
        if tab.color then
            imgui.PushStyleColor(...)
        end
        if imgui.Selectable(tab.label, selected) then
            activeKey = tab.key
        end
        if tab.color then imgui.PopStyleColor() end
    end
    imgui.EndChild()
    imgui.SameLine()

    return activeKey
end
The caller then draws the detail pane however they want:

local activeTab = lib.ui.verticalTabs(imgui, { id = "BiomeTabs", navWidth = 180, tabs = myTabs, activeKey = currentTab })
if activeTab ~= currentTab then currentTab = activeTab end

imgui.BeginChild("detail", 0, 0, false)
if currentTab == "Erebus" then
    DrawErebus(imgui, uiState)
elseif currentTab == "Oceanus" then
    DrawOceanus(imgui, uiState)
end
imgui.EndChild()
Clean. The caller owns both the state and the content. No callbacks.

lib.ui.isVisible(uiState, condition)

Kept as a helper because the structured anyOf form is not a one-liner:

-- condition can be:
-- a string alias (bool check)
-- { alias, value } (equality check)  
-- { alias, anyOf = { ... } } (multi-value check)
function lib.ui.isVisible(uiState, condition)
    if type(condition) == "string" then
        return uiState.view[condition] == true
    end
    if type(condition) ~= "table" then return true end
    local value = uiState.view[condition.alias]
    if condition.value ~= nil then return value == condition.value end
    if condition.anyOf ~= nil then
        for _, v in ipairs(condition.anyOf) do
            if value == v then return true end
        end
        return false
    end
    return value == true
end

Simplified:

function special.standaloneUI(def, store, opts)
    opts = opts or {}
    local showWindow = false

    local function renderWindow()
        if def.modpack and coordinators[def.modpack] then return end
        if not showWindow then return end
        local imgui = rom.ImGui
        if imgui.Begin((opts.windowTitle or def.name) .. "###" .. def.id) then
            local enabled = store.read("Enabled") == true
            local val, chg = imgui.Checkbox("Enabled", enabled)
            if chg then
                local ok, err = public.mutation.setEnabled(def, store, val)
                if ok and public.mutation.mutatesRunData(def) then
                    rom.game.SetupRunData()
                end
            end

            local dbgVal, dbgChg = imgui.Checkbox("Debug Mode", store.read("DebugMode") == true)
            if dbgChg then store.write("DebugMode", dbgVal) end

            if store.uiState and imgui.Button("Audit + Resync") then
                special.auditAndResyncState(def.name or def.id, store.uiState)
            end

            local drawTab = type(opts.getDrawTab) == "function" and opts.getDrawTab() or opts.drawTab
            local afterDrawTab = type(opts.getAfterDrawTab) == "function" and opts.getAfterDrawTab() or opts.afterDrawTab

            if drawTab then
                imgui.Separator()
                imgui.Spacing()
                local uiState = store.uiState
                drawTab(imgui, uiState)
                if afterDrawTab then
                    afterDrawTab(imgui, uiState)
                end
                if uiState and uiState.isDirty() then
                    local ok, err = special.commitState(def, store, uiState)
                    if ok and public.mutation.mutatesRunData(def) then
                        rom.game.SetupRunData()
                    end
                end
            end

            imgui.End()
        else
            showWindow = false
        end
    end

    local function addMenuBar()
        if def.modpack and coordinators[def.modpack] then return end
        if rom.ImGui.BeginMenu(def.name) then
            if rom.ImGui.MenuItem(def.name) then showWindow = not showWindow end
            rom.ImGui.EndMenu()
        end
    end

    return { renderWindow = renderWindow, addMenuBar = addMenuBar }
end

function store.getPackedAliases(alias)
    local aliasNode = uiState.getAliasNode and uiState.getAliasNode(alias) or nil
    if not aliasNode or not aliasNode._bitAliases then return {} end
    local result = {}
    for _, child in ipairs(aliasNode._bitAliases) do

how do standalone/framework mode render work under this plan?

Standalone mode (special.standaloneUI) — the floating window for special modules:

renderWindow becomes:

local function renderWindow()
    if not showWindow then return end
    imgui.SetNextWindowSize(...)
    imgui.Begin(windowTitle)
        -- enabled checkbox, debug checkbox, audit button, separator
        local drawTab = opts.getDrawTab and opts.getDrawTab() or opts.drawTab
        if drawTab then
            drawTab(imgui, uiState)
        end
        if opts.afterDrawTab then
            opts.afterDrawTab(imgui, uiState)
        end
        if uiState.isDirty() then
            special.commitState(def, store, uiState)
        end
    imgui.End()
end
No runPass. No def.ui fallback. No getCachedPreparedNode. Quick content path removed. The module hands Lib a drawTab function; Lib calls it and handles flush. That's the entire contract.

Framework/coordinator mode (coordinator.standaloneUI) — regular modules coordinated by the pack:

The coordinator owns the tab bar. It calls each active module's DrawTab inside BeginTabItem. The module's standaloneUI-equivalent only wires up the menu bar toggle (the enabled checkbox and debug checkbox that appear when the menu bar entry is open).

coordinator.standaloneUI simplifies to approximately:

local function renderMenuBar()
    if imgui.BeginMenu(def.name) then
        -- enabled checkbox
        -- debug checkbox (if debug mode)
        imgui.EndMenu()
    end
end
rom.gui.add_to_menu_bar(renderMenuBar)
The coordinator calls module.DrawTab(imgui, uiState) directly, then checks uiState.isDirty() and calls commitState. No runPass, no def.ui, no pipeline — just the call and the flush.

Net difference for module authors:

Before the plan: special modules pass getDrawTab getters into standaloneUI, which wraps them in runPass, which wraps them in before/draw/after lifecycle hooks, which eventually calls drawTree on def.ui if no drawTab is provided.

After the plan: module provides drawTab(imgui, uiState). Lib calls it. Module uses ImGui directly for structure. Module uses lib.widgets.* for bound controls. Done. The same function works whether the module is standalone or coordinated — the coordinator just calls it in a different context.

The regular vs special distinction survives only as a deployment detail (does the module show its own window, or does the coordinator host it). It has no effect on how DrawTab is written.
